# PROJECT_DESCRIPTION.md

## Название проекта
ChessCoachAi

## Цель проекта
Создать минималистичный веб-сервис для шахматных тренеров, который помогает быстро и удобно анализировать партии учеников с помощью ИИ.

Тренер получает:
- Автоматический импорт и анализ партий учеников из Lichess
- Детальные отчёты с выявлением повторяющихся ошибок и слабых сторон
- Статистику новых партий после последнего анализа
- Рекомендации партий для разбора на уроке

Сервис должен быть максимально похож на Lichess по стилю (тёмная тема, цвета, шрифты, отступы, карточки, кнопки), но проще и сосредоточен только на коучинге.

## Текущий технологический стек MVP (март 2026)

- Frontend: Next.js 15 (App Router), TypeScript, Tailwind CSS + shadcn/ui
- Шахматная доска: react-chessground (или react-chessboard) — максимально близко к доске Lichess
- Аутентификация и база данных: Supabase (Auth + Postgres + RLS + Realtime)
- Авторизация пользователей: Lichess OAuth 2.0 (через Supabase Auth)
- Анализ позиций: Lichess Cloud Eval API (Stockfish в облаке Lichess, с токеном тренера)
- LLM для комментариев и отчётов: Gemini Flash Lite (Google AI Studio) / DeepSeek / Ollama Cloud — выбор по ключу тренера
- Хостинг фронтенда: Cloudflare Pages (бесплатно навсегда)
- Хостинг backend-логики (если потребуется Python): Render.com Free tier или Railway

## Основные экраны MVP

### 1. Авторизация
- Единственный способ входа: кнопка «Войти через Lichess»
- После входа сохраняется access_token тренера в Supabase (для использования Lichess Cloud Eval от его имени)

### 2. Главная страница — «Мои ученики»
- Большое поле ввода для массового добавления ников Lichess
- При вводе ника и нажатии Enter/кнопки «Добавить» — ник превращается в removable chip (тег с крестиком)
- Ниже — список учеников (карточки или строки таблицы):
  - Никнейм
  - Дата последнего анализа
  - Количество новых партий (с разбивкой по блиц/рапид/буллит)
- Клик по ученику → открывается его профиль в основной области

### 3. Профиль ученика
- Заголовок с ником ученика и текущим рейтингом (подтягивается из Lichess)
- Блок «Анализ новых партий»:
  - Выбор типа: Bullet / Blitz / Rapid
  - Поле «Сколько последних партий анализировать» (по умолчанию 10–20)
  - Чекбоксы: Белые / Чёрные / Выигранные / Проигранные
  - Большая кнопка «Проанализировать» (основной акцентный цвет Lichess)
- Блок «Партии для подробного анализа» — список 3–5 рекомендованных партий с:
  - Коротким описанием (почему стоит разобрать)
  - Ссылкой на партию в Lichess или встроенной доской

### 4. Настройки (боковая панель или выпадающее меню вверху)
- Раздел «Мои API-ключи»:
  - OpenRouter API Key (поле ввода + кнопка «Сохранить»)
  - Google AI Studio (Gemini) API Key
  - Ollama Cloud API Key / Endpoint
- Ключи сохраняются зашифрованно в Supabase и используются при анализе от имени тренера

## Общие требования к дизайну и UX
- Полностью тёмная тема в стиле Lichess (#1f1f1f фон, #2a2a2a карточки, акцент #4fc3f7, текст #e0e0e0)
- Левая фиксированная боковая панель (как в Lichess)
- Верхняя панель с логотипом «ChessCoach» и именем тренера
- Минимализм, много свободного пространства, чистые линии, скруглённые углы, hover-эффекты как в Lichess
- Полностью responsive (отлично работает на мобильных устройствах и планшетах)
- Все надписи и интерфейс на русском языке

## Технические приоритеты
- Использовать Lichess OAuth для авторизации тренера
- Использовать Lichess Cloud Eval с токеном конкретного тренера (чтобы лимиты были персональными)
- LLM-промпты должны быть подробными, мотивирующими, с примерами вариантов и планом тренировок на русском
- Отчёты сохраняются в Supabase для быстрого доступа и истории
- Поддерживать будущий рост: подписки, уведомления, графики прогресса, интеграция с Telegram/VK
