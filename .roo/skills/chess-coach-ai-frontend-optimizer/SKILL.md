# Когда использовать этот skill
Используйте этот skill при разработке, доработке или интеграции фронтенд-компонентов и страниц для проекта ChessCoachAi. Он оптимизирован для работы с дизайнами из v0.dev и соблюдения стиля Lichess.

# Когда НЕ использовать этот skill
- При работе над backend-логикой, не связанной с API-клиентами Supabase или интеграцией UI.
- При разработке функционала, выходящего за рамки MVP (если нет явного указания).

# Основные правила и поведение
- **Стек:** Next.js 15 App Router, TypeScript, Tailwind CSS, shadcn/ui.
- **Vercel & Next.js Compliance (Strict):**
  - Всегда используй `use client` или `use server` директивы.
  - **Никаких `any`**: используй строгую типизацию TypeScript.
  - Избегай `console.log` в финальном коде.
  - Все внешние ссылки: `rel="noopener noreferrer"`.
  - Обязательное использование `next/image` (с `width`/`height` или `fill`) и `next/font`.
  - Обработка ошибок: `try/catch` в Server Actions, `error.tsx` для роутов.
  - Секреты: только через `process.env`. Клиентские ключи — строго с `NEXT_PUBLIC_`.
  - Middleware: только Edge Runtime, минимальный размер.
  - Оптимизация сборки: избегай тяжелых библиотек, предпочитай нативные решения или легкие аналоги.
- **Интеграция v0.dev:**
  - Проверяй наличие кода в `/v0design/` перед генерацией UI с нуля.
  - Адаптируй и пересобирай фронтенд на основе существующих дизайнов.
- **Стиль Lichess (Dark Mode):**
  - Фон: `#1f1f1f`
  - Карточки: `#2a2a2a`
  - Акцент: `#4fc3f7`
  - Текст: `#e0e0e0`
  - Скругленные углы, hover-эффекты как в Lichess, много свободного пространства.
- **Локализация:** Весь интерфейс (labels, placeholders, кнопки, ошибки) — только на **русском языке**.
- **Приоритет MVP-экранов:**
  1. Авторизация (кнопка «Войти через Lichess»).
  2. Главная «Мои ученики» (ввод ников → chips, список карточек).
  3. Профиль ученика (анализ партий, фильтры, список рекомендаций).
  4. Настройки (формы для API-ключей: OpenRouter, Gemini, Ollama).
- **Supabase:** Используй `@supabase/ssr` (`createServerClient` / `createBrowserClient`), соблюдай RLS, не раскрывай ключи в UI.
- **Шахматная доска:** Используй `react-chessground` ��ли `react-chessboard` с тёмной темой Lichess.
- **Оптимизация токенов:**
  - Генерируй только необходимый код (минимум boilerplate).
  - Используй diff/patch вместо полной перезаписи файлов при доработке.
  - Проверяй наличие компонентов в `shadcn/ui` перед созданием кастомных.
  - Держи код компонента/страницы в пределах 150 строк.

# Примеры (Input -> Output)

**Input:** "Интегрируй сгенерированный в v0 компонент для главной страницы из v0design/main-page.tsx"
**Output:** (Агент читает файл, извлекает JSX, адаптирует под структуру App Router, заменяет стандартные цвета на Lichess-палитру и переводит текст на русский).

**Input:** "Создай страницу настроек с формами для API-ключей в стиле Lichess"
**Output:** (Агент генерирует страницу с использованием shadcn/ui Card и Input, применяет цвета #1f1f1f/#2a2a2a, добавляет обработку сохранения через Supabase client).

**Input:** "Адаптируй chips для добавления учеников под русский язык и тёмную тему"
**Output:** (Агент обновляет компонент Badge или кастомный Chip: `variant="outline"`, фон `#2a2a2a`, текст `#e0e0e0`, кнопка удаления с подсказкой "Удалить").

# Валидация
- Код всегда responsive и в dark mode.
- В UI отсутствует английский язык.
- Используется только стек из `PROJECT_DESCRIPTION.md`.
- Генерируемый код компактен (< 150 строк на файл).
- Отсутствие ошибок типизации (TypeScript strict mode).
- Код готов к деплою на Vercel/Cloudflare без правок (production-ready).
