# Список исправлений и приоритетных задач (FIXME)

Этот документ содержит список выявленных проблем и план действий по их устранению.

## 🔴 Критические (Исправить немедленно)

### 1. Анонимная авторизация Supabase
- **Проблема:** В `src/app/auth/callback/route.ts` используется `signInAnonymously()`. Это приводит к тому, что при очистке кук или истечении сессии пользователь теряет доступ к своему профилю (создается новый `user.id`).
- **Решение:**
    1.  Запросить профиль Lichess через `https://lichess.org/api/account` после получения токена.
    2.  Использовать Lichess ID как уникальный идентификатор для входа в Supabase (через кастомный JWT или привязку Lichess ID к метаданным пользователя).
    3.  Либо реализовать полноценный OAuth Supabase провайдер (если это возможно с Lichess).

### 2. Отсутствие обработки истечения токена
- **Проблема:** Lichess OAuth токены имеют ограниченный срок действия. Код сохраняет `refresh_token`, но механизм обновления в middleware или действиях (actions) пока отсутствует.
- **Решение:** Добавить логику проверки валидности токена перед запросами к Lichess API и автоматический рефреш.

## 🟡 Средний приоритет (Улучшения)

### 3. Переход на Tailwind v4 в дизайне
- **Проблема:** Компоненты в `v0design` написаны на Tailwind v3 (классы `bg-primary/15`, `text-muted-foreground` и т.д.). В основном проекте Tailwind v4 может требовать иного способа прокидывания этих переменных из `globals.css`.
- **Решение:** Адаптировать `globals.css` в `chess-coach-ai`, чтобы он корректно определял цвета темы (primary, accent, muted) в формате CSS Variables, который понимает v4.

### 4. Интеграция `useApp` контекста
- **Проблема:** Дизайн дашборда завязан на контекст `useApp`, который пока не реализован в основном проекте.
- **Решение:** Создать `AppContext` в `src/lib/context` (или использовать TanStack Query для управления состоянием), который будет синхронизироваться с Supabase, а не хранить данные только локально.

## 🟢 Низкий приоритет (Бэклог)

### 5. Локализация интерфейса
- **Проблема:** В `v0design` часть текста на русском, часть на английском.
- **Решение:** Привести весь UI к единому русскому языку согласно `PROJECT_DESCRIPTION.md`.

### 6. Обработка Rate Limits Lichess API
- **Проблема:** При массовом импорте партий для многих учеников можно упереться в лимиты Lichess API (429 Too Many Requests).
- **Решение:** Добавить очередь запросов с задержкой (throttling/debouncing).
